Hops

In Task 1, we used the traceroute command to trace the route from our machine to the assigned website, which is chideraadaeze.github.io in this case. Traceroute provides information about the number of hops or network nodes between our machine and the destination website, helping us understand the path that data packets take to reach the destination.

The output of the traceroute command shows that there are multiple network nodes (hops) between our machine and the website. It lists the IP addresses of these nodes, along with the round-trip time (in milliseconds) it takes for packets to travel to each node.

Notably, some hops did not respond (denoted by '*'), which can happen due to firewall or security configurations that prevent response to traceroute requests.

Overall, traceroute provides valuable insights into the network path to the website, helping us identify the number of hops, response times, and potential bottlenecks in the route.
